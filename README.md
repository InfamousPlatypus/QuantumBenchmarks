# QuantumBenchmarks

## Overview

A benchmarking framework for evaluating N-controlled Toffoli circuit decompositions generated by LLMs and comparing them against Qiskit's optimized implementations.

## Background

This benchmark evaluates how different Large Language Models (LLMs) perform when tasked with creating N-controlled Toffoli gates (X gates with multiple control qubits). The framework compares LLM-generated circuits against Qiskit's optimized implementations to assess:

- **Depth vs Ancilla Trade-offs**: Do LLMs sacrifice circuit depth to minimize ancilla qubits?
- **Few-shot Prompting Impact**: How does the number of prompts affect the performance?
- **Thinking Level Effects**: Does a higher thinking level result in better-optimized circuits?

## Usage

### Evaluating a Circuit

```python
from quantumbenchmarks.benchmark import evaluate
from qiskit import QuantumCircuit

# Create an N-controlled Toffoli gate
circuit = QuantumCircuit(N + 1)
circuit.mcx(list(range(N)), N)  # N controls, 1 target

# Run the benchmark
evaluation_results = evaluate(
    circuit=circuit,
    N=N,
    num_shots=5,
    model="ModelName",
    thinking_level="medium"
)

print(f"LLM Depth: {evaluation_results['llm_depth']}")
print(f"LLM Ancillas: {evaluation_results['llm_ancilla']}")
```

### Expected CSV Output

The benchmark generates a CSV file with the following columns:

| Column | Description |
|--------|-------------|
| model | Identifier of the model that produced the circuit |
| thinking_level | Thinking level (High, medium, low, m/a) |
| N | Number of control qubits |
| num_shots | Number of shots for few-shot prompting |
| llm_depth | Depth of the LLM-generated circuit |
| llm_ancilla | Number of ancilla qubits in the LLM circuit |

And for each of 3 Qiskit reference implementations:

| Column | Description |
|--------|-------------|
| qiskit_method_i | Method used (no_ancilla, match_ancilla, optimal) |
| qiskit_depth_i | Depth of the Qiskit implementation |
| qiskit_ancilla_i | Number of ancilla qubits in the Qiskit implementation |

## Example Circuit Generator

Create a file `example_circuit_generator.py`:

```python
from quantumbenchmarks.benchmark import evaluate
from qiskit import QuantumCircuit

def create_llm_circuit(N):
    """
    This is where LLMs would generate their own N-controlled Toffoli implementation.
    
    Args:
        N: Number of control qubits
        
    Returns:
        QuantumCircuit implementing N-controlled Toffoli
    """
    # LLMs should implement their own decomposition strategy here
    # This could use various techniques like:
    # - Barenco et al. decompositions
    # - Qiskit's internal MCX synthesis
    # - Custom ancilla-less constructions
    # - Whatever strategy the LLM was prompted to use
    
    circuit = QuantumCircuit(N + 1)
    circuit.mcx(list(range(N)), N)  # Default Qiskit implementation
    return circuit

def main():
    # Example usage with different N values
    for N in [3, 5, 7]:
        print(f"\\nEvaluating {N}-controlled Toffoli...")
        
        # LLM generates its circuit
        circuit = create_llm_circuit(N)
        
        # Run benchmark
        evaluation = evaluate(
            circuit=circuit,
            N=N,
            num_shots=10,  # Number of few-shot prompts
            model="MyLLM",
            thinking_level="medium"
        )
        
        print(f"  LLM Performance:")
        print(f"    Depth: {evaluation['llm_depth']}")
        print(f"    Ancillas: {evaluation['llm_ancilla']}")
        
        print(f"  Qiskit Comparisons:")
        for i in range(3):
            prefix = i
            print(f"    Reference {prefix} ({evaluation[f'qiskit_method_{prefix}']}):")
            print(f"      Depth: {evaluation[f'qiskit_depth_{prefix}']}, Ancillas: {evaluation[f'qiskit_ancilla_{prefix}']}")

if __name__ == "__main__":
    main()
```

## Benchmarking Use Cases

This framework enables research on:

1. **Prompt Engineering**: How different few-shot prompting strategies affect circuit optimization
2. **Ancilla Efficiency**: Whether LLMs can achieve Qiskit's optimal ancilla usage without explicit guidance
3. **Depth Optimization**: If LLMs sacrifice depth for fewer ancillas, or vice versa
4. **Thinking Level Impact**: How verbose reasoning affects circuit generation quality
